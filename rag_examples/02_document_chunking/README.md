## 📖 学习指导

### 为什么需要切片？

```
不切片的后果：
  整篇文章作为一个 chunk → 检索时混入大量无关信息 → LLM 被干扰 → 回答不准

切太大的后果：
  一个 chunk 包含多个主题 → 检索命中时附带大量噪声 → LLM 分不清重点

切太小的后果：
  语义不完整 → 检索时找不到相关内容 → 检索召回率为 0

好的切片 = RAG 系统效果好坏的决定性因素之一
```

### 第 1 步：固定规则切片（01_fixed_chunking.py）

**本节讲什么：**
- 最简单的切片方法——按固定规则切割，不考虑语义边界
- 3 种切割规则：按字符数 / 按段落 / 按句子
- 带重叠的固定切片（减少语义切断的影响）

**核心概念：**

```
3 种固定切分规则：

按字符数：每 N 个字符一切
  "人工智能是计算机科学的一个分支..." → [chunk1:100字] [chunk2:100字] ...
  ✓ 最简单    ✗ 可能在句子中间切断

按段落：每个空行分隔的段落一切
  "段落1\n\n段落2\n\n段落3" → [段落1] [段落2] [段落3]
  ✓ 保持结构完整    ✗ 段落长度可能不均

按句子：每个句号/问号/感叹号一切
  "今天天气很好。我们去公园吧！" → ["今天天气很好", "我们去公园吧"]
  ✓ 语义相对完整    ✗ 句子长度差异大
```

**关键代码：**

```python
# 按字符数切片
chunks = [text[i:i+chunk_size] for i in range(0, len(text), chunk_size)]

# 按段落切片
chunks = [p.strip() for p in text.split('\n\n') if p.strip()]

# 按句子切片
chunks = re.split(r'[.!?。！？]', text)

# 带重叠切片（减少语义切断）
step = chunk_size - overlap  # 步长 = 窗口 - 重叠
```

**参数建议：**

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| chunk_size | 200-500 | 太小语义不完整，太大包含噪声 |
| overlap | 20-50 | 约 10-25 个中文字，保持上下文连续 |

### 第 2 步：滑动窗口切片（02_sliding_window.py）

**本节讲什么：**
- 固定大小的窗口在文本上滑动，相邻切片有重叠
- 3 个核心参数：窗口大小（window_size）、步长（step_size）、重叠率
- 4 种窗口变体：基础字符窗口 / 单词窗口 / 句子边界窗口 / 参数对比实验

**核心概念：**

```
滑动窗口原理：

文本: ABCDEFGHIJKLMNOPQRSTUVWXYZ
      |----窗口100----|          ← 第1个切片 [0:100]
              |----窗口100----|  ← 第2个切片 [50:150]，重叠50字符
                      |----窗口100----| ← 第3个切片 [100:200]

关键公式：
  重叠字符数 = window_size - step_size
  重叠率 = (window_size - step_size) / window_size × 100%
```

**参数对比实验：**

| 步长 | 重叠率 | 切片数量 | 适用场景 |
|------|--------|---------|---------|
| 80（小重叠） | 20% | 少 | 节省空间，粗略检索 |
| 50（中重叠） | 50% | 中 | **推荐默认**，平衡性能和精度 |
| 20（大重叠） | 80% | 多 | 法律/医疗等精细检索 |

**不同文档长度的建议配置：**

| 文档长度 | 窗口大小 | 步长 | 重叠率 |
|---------|---------|------|--------|
| 短文档（<1000 字） | 200 | 100 | 50% |
| 中档文档（1000-5000 字） | 500 | 250 | 50% |
| 长文档（>5000 字） | 1000 | 500 | 50% |
| 高精度需求 | 300 | 75 | 75% |

**滑动窗口 vs 固定切片：**

```
滑动窗口优势：                    滑动窗口劣势：
  ✓ 保持上下文连续性                ✗ 数据冗余（重叠部分重复存储）
  ✓ 减少语义切断的影响              ✗ 切片数量更多
  ✓ 检索时召回更完整的信息          ✗ 检索时可能有重复结果
```

### 第 3 步：AI 辅助切片（03_ai_chunking.py）

**本节讲什么：**
- 利用 LLM 识别语义边界，按主题自然分段
- 3 种实现方式：真实 LLM API / 模拟启发式 / 句子聚类
- LangChain 文本分割器（RecursiveCharacterTextSplitter）
- AI 切片的最佳实践和成本控制

**核心概念：**

```
AI 切片工作流程：

1. 将文档发送给 LLM
2. LLM 分析内容，识别主题/语义边界
3. LLM 返回建议的切分点（JSON 格式）
4. 按切分点分割文档

Prompt 示例：
  "请分析以下文本，找出 5 个语义边界（最适合切分的位置）。
   按 JSON 格式返回切分点和切分原因。"
```

**3 种 AI 切片方式对比：**

| 方式 | 实现 | 质量 | 成本 | 适用 |
|------|------|------|------|------|
| 真实 LLM API | 调用 qwen-plus 返回切分点 | 优秀 | API 费用 | 高质量 RAG |
| 模拟启发式 | 按段落 + 大小限制合并 | 良好 | 无 | 教学演示 |
| 句子聚类 | 固定句子数聚合 | 中等 | 无 | 轻量场景 |

**推荐流程（混合方法）：**

```
原始文档
    │
[预处理] 清理格式、统一编码
    │
[粗切] 按段落/章节分割（规则方法，免费）
    │
[精切] 对每个粗切片段用 AI 识别语义边界
    │
[后处理] 合并过小切片，过滤空切片
    │
最终切片列表

成本：只用少量 API 调用，而非处理全文
```

**LangChain 分割器：**

```python
# 字符分割器（按分隔符切）
splitter = CharacterTextSplitter(
    separator="\n", chunk_size=100, chunk_overlap=20
)

# 递归字符分割器（按多级分隔符切）
splitter = RecursiveCharacterTextSplitter(
    separators=["\n\n", "\n", "。", ".", " "],
    chunk_size=100, chunk_overlap=20
)
# 优先按段落分，分不开再按句子分，再分不开再按空格分
```

### 第 4 步：概要生成切片（04_summary_chunking.py）

**本节讲什么：**
- 为每个切片生成一句话摘要
- 用摘要进行检索（减少噪声），用原文进行回答
- 混合检索：摘要 + 原文加权融合
- 完整 RAG 流程演示（摘要检索 + LLM 生成回答）

**核心概念：**

```
为什么需要摘要切片？

直接检索原文的问题：
  用户问"机器学习有几种学习方式？"
  → 匹配到一个 500 字的 chunk
  → chunk 中只有 50 字与问题相关
  → 其余 450 字是噪声，干扰 LLM 理解

摘要切片的解决：
  检索时：匹配简洁的摘要（~30 字，核心信息）
  回答时：使用完整的原文（~500 字，完整上下文）
```

**完整流程：**

```
1. 文档切片 → 得到 [chunk1, chunk2, chunk3, ...]
2. 生成摘要 → 得到 [summary1, summary2, summary3, ...]
3. 用户提问 → 问题与摘要匹配 → 找到最相关的 top-k 摘要
4. 通过摘要索引 → 获取对应的原文 chunk
5. 将原文 + 问题 → 交给 LLM 生成回答
```

**混合检索策略：**

```python
# 摘要检索得分 + 原文检索得分 = 最终得分
combined_score = 0.6 × summary_score + 0.4 × chunk_score
# 摘要权重更高（0.6），因为它更简洁精准
# 原文权重（0.4），保留细节匹配能力
```

**何时使用摘要切片：**

| 文档长度 | 建议 | 原因 |
|---------|------|------|
| <500 字 | 直接用原文 | 简单高效，无需额外处理 |
| 500-2000 字 | 摘要 + 原文混合检索 | 平衡效果和成本 |
| >2000 字 | 用摘要检索 | 减少噪声干扰 |

## ⚠️ 常见问题

### Q: 切片大小怎么选择？
A: 从 300-500 字符开始。太小语义不完整，太大包含噪声。可以用 `05_chunking_comparison.py` 对比不同大小的效果。

### Q: 为什么要重叠？不重叠不行吗？
A: 不重叠的话，如果一句话刚好被切成两半，检索时可能两边都匹配不到完整信息。重叠能减少这种"语义切断"的问题。

### Q: AI 切片一定比固定规则好吗？
A: 不一定。对于结构清晰的文档（如已分好段落），固定段落切片效果就很好。AI 切片适合结构混乱或需要精细切分的场景，但成本更高。

### Q: 我的文档是 PDF/Word 格式的怎么办？
A: 先转成纯文本（TXT）再切片。PDF 需要先提取文字（可用 `PyPDF2` 或 `pdfplumber`），Word 用 `python-docx`。
