# 03_retrieval_methods - 检索方法

本模块演示 RAG 系统中的核心检索方法。

## 检索方法的重要性

检索是 RAG 系统的第一步，也是至关重要的一步：
- **检索不准**：后续 LLM 回答再强也没用（垃圾进，垃圾出）
- **检索精准**：LLM 可以基于相关信息给出高质量回答

## 检索方法对比

| 方法 | 原理 | 优点 | 缺点 | 适用场景 |
|------|------|------|------|----------|
| 标量查询 | SQL 条件筛选 | 精确、快速 | 无法语义匹配 | 有明确过滤条件 |
| 向量检索 | 向量相似度 | 语义匹配 | 需要 Embedding | 语义搜索 |
| 关键字检索 | BM25 算法 | 精确匹配关键词 | 无法理解语义 | 专有名词检索 |
| 混合检索 | 多种方法结合 | 综合优势 | 实现复杂 | 生产环境 |
| Rerank | 重排序 | 提升精度 | 额外计算成本 | 高精度场景 |

## 核心理论

### 1. 向量检索原理

```
用户问题 → Embedding → 问题向量
                        ↓
            与库中向量逐一比较相似度
                        ↓
            返回最相似的 Top-K 文档
```

**相似度计算公式：**
- 余弦相似度：`cos(θ) = A·B / (‖A‖ × ‖B‖)`
- 值域：[-1, 1]，越接近 1 越相似

### 2. BM25 算法原理

BM25 (Best Matching 25) 是经典的关键词检索算法：

```
Score(Q, D) = Σ [IDF(qi) × (f(qi,D) × (k1+1)) / (f(qi,D) + k1 × (1-b+b×|D|/avgdl))]
```

简化理解：
- **TF（词频）**：词在文档中出现越多越重要
- **IDF（逆文档频率）**：在越少文档中出现的词越重要
- **文档长度归一化**：长文档中的词频需要打折

### 3. 混合检索策略

```
用户问题
    ↓
┌───────────────┬───────────────┐
│   向量检索     │  关键字检索    │
│  (语义匹配)    │  (精确匹配)    │
└───────┬───────┴───────┬───────┘
        ↓               ↓
    结果 A           结果 B
        ↓               ↓
    ┌──┴───────────────┴──┐
    │    融合 + 重排序      │
    └──────────┬──────────┘
               ↓
         最终结果
```
