## 为什么需要评估？

不评估 = 盲目飞行：
- 换了 Embedding 模型 → 效果变好还是变差？不知道
- 调整了切片参数 → 检索质量有没有提升？不知道
- 上线了新版本 → 回答质量下降了吗？不知道

**评估是 RAG 系统迭代优化的基础。**

## RAGAS 四大指标

| 指标 | 含义 | 评估对象 | 满分含义 |
|------|------|----------|----------|
| **Faithfulness（忠实度）** | 回答是否基于检索到的上下文 | LLM 生成 | 没有编造内容 |
| **Answer Relevancy（答案相关性）** | 回答是否回答了问题 | LLM 生成 | 完全回答问题 |
| **Context Precision（上下文精确率）** | 检索结果中相关文档的占比 | 检索器 | 所有结果都相关 |
| **Context Recall（上下文召回率）** | 相关文档被检索到的比例 | 检索器 | 没有遗漏相关内容 |
